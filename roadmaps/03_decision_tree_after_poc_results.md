# 03 — Decision Tree After POC Results

Use this after the POC. Do not proceed linearly if results indicate a pivot.

## If governance reviewers love the receipt but not the graph details

Next move:

- Build committee report generator.
- Hide graph complexity behind drill-down.
- Prioritize audit artifact packaging.

Do not:

- Overbuild graph UI.

## If reviewers love next-best-input recommendations

Next move:

- Invest in EvidenceFlow result distributions.
- Build candidate-input library.
- Add outcome/posture-shift validation.

Do not:

- Expand role swarm prematurely.

## If AI-provenance depth gets strongest reaction

Next move:

- Build AI-provenance census dashboard.
- Focus on source lineage and false-consensus risk.
- Build prospective receipt capture.

Do not:

- Lead with clinical content.

## If role agents are noisy

Next move:

- Reduce role set.
- Use deterministic/curated role outputs.
- Improve schemas and rejection rules.
- Build ablation tests.

Do not:

- Add more agents.

## If graph outputs seem arbitrary

Next move:

- Build calibration workbench.
- Add expert-adjustable parameters.
- Show sensitivity ranges.
- Use simple deterministic rules first.

Do not:

- Hide uncertainty behind precision.

## If EvidenceFlows cannot produce result distributions

Next move:

- Manually curate result distributions for narrow candidate-input libraries.
- Use evidence tools only for guideline support.
- Delay open-ended VOI automation.

Do not:

- Pretend narrative evidence summaries are enough for VOI.

## If users say this is too abstract

Next move:

- Pick one visible governance wedge: AI-provenance census, next-best-input review, or decision-warrant receipt.
- Use multiple artifact examples but one buyer story.

Do not:

- Claim universal platform before concrete value.

## If users say this is clinically useful but not governable

Next move:

- Explore clinical workflow product later.
- Keep POC as research validation.

Do not:

- Jump to live alerts without safety case.

## If users say this is valuable but they cannot buy it

Next move:

- Explore adjacent buyers: risk/legal, payer/utilization, AI vendors needing assurance, malpractice carriers, certification consultants.

Do not:

- Assume governance budget exists just because governance need exists.
